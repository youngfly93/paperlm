# paperlm v0.1.2

_In development._

## Highlights

- Generate `paperlm_dict`, `paperlm_json`, and `paperlm_chunks_jsonl`
  lazily on first access instead of during every conversion.
- Keep `result.markdown`, `result.ir`, and `result.engine_used`
  immediately available for existing callers.
- Add a release-facing benchmark report entrypoint that writes
  `benchmarks/reports/latest.md` and `benchmarks/reports/latest.json`.
- Add GitHub Actions quality gates for lint, typecheck, fast tests,
  source/wheel build, and package metadata validation.

## Notes

- Batch and worker-pool paths still materialize sidecars when they need
  JSONL output or artifact files.
- `make benchmark-report` defaults to the existing 8-fixture corpus and
  the core triad: paperlm, MarkItDown baseline, and Docling standalone.
- Heavy Docling/OCR integration tests remain manual opt-in in CI.
