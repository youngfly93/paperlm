# paperlm v0.1.1

_Planned patch release._

## Highlights

- Add `paperlm-tools warmup` for deployment-time model initialization.
- Warmup runs selected engines through guarded worker subprocesses with
  timeout and hard RSS limits.
- Warmup validates that the requested engine actually ran, so missing
  optional extras cannot silently pass through fallback.

## Usage

```bash
paperlm-tools warmup --engine docling
paperlm-tools warmup --engine docling,ocr --timeout-s 600 --max-rss-mb-hard 6144
```

## Notes

- `docling` warmup requires `pip install paperlm[docling]`.
- `ocr` warmup requires `pip install paperlm[ocr]`.
- `fallback` warmup is lightweight and uses the core pdfminer path.
